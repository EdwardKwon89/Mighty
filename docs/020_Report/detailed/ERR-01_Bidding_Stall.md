# Audit Report: [ERR-01] Bidding Phase Stall

## 1. 개요
게임 시작 후 비딩(공약) 단계에서 특정 플레이어(특히 봇)의 차례에서 게임이 더 이상 진행되지 않고 멈추는 현상 발생.

## 2. 원인 분석
- **타이머 로직 미흡**: `startTimer` 호출 시 봇의 차례를 체크하는 `checkBotTurn` 로직이 누락되어 봇이 행동을 개시하지 않음.
- **인덱스 계산 오류**: `currentBidderIndex`가 `isPassed` 상태인 플레이어를 건너뛰는 과정에서 무한 루프 또는 잘못된 인덱스를 참조할 가능성 존재.
- **클라이언트-서버 동기화**: 서버는 비딩 완료로 판단했으나 클라이언트에 상태 변화가 전달되지 않아 UI가 갱신되지 않음.

## 3. 조치 방법
- **봇 행동 트리거 통합**: `startTimer` 함수 내부에 `checkBotTurn`을 명시적으로 호출하여 봇의 차례가 오면 즉시(딜레이 후) 행동하도록 수정.
- **견고한 인덱스 탐색**: 비딩 패스 여부를 확인하며 다음 비딩자를 찾는 `while` 루프에 `safeGuard`를 추가하여 무한 루프 방지.
- **자동 비딩 폴백**: 정해진 시간 내에 비딩이 이루어지지 않을 경우 서버 사이드에서 자동으로 `Pass` 처리를 수행하는 타임아웃 핸들러 강화.

## 4. 검증 결과
- 5인 봇 시뮬레이션 결과, 모든 플레이어가 정상적으로 비딩을 마치고 다음 단계(Exchange)로 전이됨을 확인.
