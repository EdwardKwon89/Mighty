# Audit Report: [ERR-02] Bot Play Logic Failure

## 1. 개요
카드 플레이 단계에서 봇이 유효하지 않은 카드를 내려고 시도하거나, 적절한 카드를 선택하지 못해 게임이 멈추는 현상.

## 2. 원인 분석
- **엄격한 유효성 검사**: 엔진의 `canPlayCard` 규칙이 봇의 단순 선택 로직보다 엄격하여 봇이 제안한 카드가 서버에서 거절됨.
- **예외 처리 부재**: 카드가 거절되었을 때 봇이 다른 카드를 시도하는 로직이 없어 해당 턴에서 멈춤.
- **상태 미갱신**: 봇의 카드 제출 실패 후 타이머가 재시작되지 않아 타임아웃까지 대기하게 됨.

## 3. 조치 방법
- **3단계 Failsafe 로직 도입**:
    1. **Primary**: 엔진 기반 전략적 카드 선택.
    2. **Failsafe**: 전략적 선택 실패 시, 현재 패에서 플레이 가능한 '아무 유효한 카드' 탐색 및 제출.
    3. **Fatal Fallback**: 유효성 검사 로직 자체 오류 시에도 게임 진행을 위해 무조건 첫 번째 카드를 강제 제출하고 턴을 강제로 넘김.
- **실시간 로깅**: 봇의 사고 과정(`[BOT_THINK]`)과 재시도 과정(`[BOT_RETRY]`)을 서버 로그에 남겨 추적 가능하게 함.

## 4. 검증 결과
- 테스트 결과, 비정상적인 상황(낼 카드가 없는 카드 구성 등)에서도 시스템이 Fatal Fallback을 통해 10트릭을 모두 완주함.
