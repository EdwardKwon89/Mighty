# Audit Report: [ERR-03] User Sync & Reconnection Issue

## 1. 개요
브라우저 새로고침이나 네트워크 순단 현상 발생 시, 플레이어가 본인의 턴임을 인지하지 못하거나 게임에서 완전히 이탈되는 현상.

## 2. 원인 분석
- **Socket ID 의존성**: 서버가 플레이어를 Socket ID로 식별하여, 새로고침 시 소켓 ID가 변경되면 기존 플레이어와 매칭되지 않음.
- **상태 복구 누락**: 재접속 시 현재 게임판의 전체 상태(`game-state`)를 즉시 전송받지 못해 UI가 초기화됨.
- **턴 표시 불일치**: 서버의 `turnIndex`와 클라이언트의 `isMyTurn` 계산 방식이 상이하여 활성화되지 않음.

## 3. 조치 방법
- **닉네임 기반 식별 (Persistent ID)**: Socket ID 대신 DB의 닉네임을 유니크 키로 사용하여 재접속 시에도 동일 플레이어로 인식하도록 `addPlayer` 로직 수정.
- **재접속 타이머 및 상태 유지**: 플레이어 이탈 시 즉시 방에서 삭제하지 않고 60초간 대기하며, 재접속 시 기존 카드 덱과 상태를 그대로 유지.
- **Full State Broadcast**: 소켓 연결 및 룸 참여 즉시 현재 룸의 모든 상태를 담은 `game-state`를 브로드캐스트하여 UI 즉시 복구.

## 4. 검증 결과
- 새로고침 테스트 결과, 약 1~2초 내에 이전 게임 상태(내 카드, 현재 트릭, 내 턴 여부)가 완벽히 복구됨을 확인.
