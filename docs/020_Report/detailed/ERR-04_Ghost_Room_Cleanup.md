# Audit Report: [ERR-04] Ghost Room & Test Data Cleanup

## 1. 개요
테스트 시뮬레이션 후 생성된 방들이 사라지지 않고 로비에 남아 실제 유저의 참여를 혼란스럽게 하거나 서버 메모리를 점유하는 현상.

## 2. 원인 분석
- **Graceful Shutdown 부재**: 테스트 에이전트 종료 시 방을 명시적으로 삭제하지 않음.
- **빈 방 유지 로직**: 플레이어가 모두 나갔음에도 불구하고 룸 객체가 Map에서 제거되지 않음.
- **테스트 접두사 필터링 누락**: 로비 리스트에서 테스트용 방(`PRIME_FIELD_TEST_`)을 구분하지 않고 모두 표시함.

## 3. 조치 방법
- **Nuclear Cleanup 인터벌**: 서버사이드에서 1초마다 `PRIME_FIELD_TEST_` 접두사를 가진 방을 강제로 삭제하는 가비지 컬렉션 로직 추가.
- **초기화 부팅**: 서버 시작 3초 후 모든 기존 룸을 `clear()` 하여 깨끗한 상태로 시작하도록 보장.
- **로비 필터링**: `broadcastRoomsList` 호출 시 테스트용 방은 클라이언트에 전송되지 않도록 필터링 처리.

## 4. 검증 결과
- 서버 재시작 및 테스트 종료 후 로비 대기실에 유령 방이 더 이상 나타나지 않음을 확인.
